การเรียกดูข้อมูลโดยใช้คิวรี่ย่อย (Subquery)

ความรู้เบื้องต้น

วัตถุประสงค์ในการทดลอง

  1. ผู้เรียนสามารถใช้การเรียกดูข้อมูลโดยใช้คิวรี่ย่อยเพื่อกำหนดคอลัมน์ของคำสั่ง SELECT ได้
  2. ผู้เรียนสามารถใช้คิวรี่ย่อยในประโยค WHERE ได้
  3. ผู้เรียนสามารถใช้คิวรี่ย่อยในประโยค HAVING ได้
  4. ผู้เรียนสามารถใช้ Correlated Subquery ได้

ทฤษฎีก่อนการทดลอง

Subquery

    คือ ชุดคำสั่งที่ซ้อนอยู่ในชุดคำสั่ง ซึ่งชุดคำสั่งที่ซ้อนอยู่ข้างในคิวรี่หลักจะถูกเรียกว่าคิวรี่ย่อย คิวรี่ย่อยจะเป็นชุดคำสั่ง SELECT เท่านั้น โดยคิวรี่ย่อยสามารถนำไปใช้กับคิวรี่หลัก SELECT INSERT UPDATE และ DELETE ได้ ซึ่งจะใช้ผลลัพธ์ของคิวรี่ย่อยเป็นเงื่อนไขในการกระทำของคิวรี่หลัก ในที่นี้จะยกตัวอย่างการใช้งานกับคำสั่ง SELECT เท่านั้น โดยผู้เรียนสามารถนำไปประยุกต์ใช้กับคำสั่งอื่นๆได้

ขั้นตอนการทดลอง

การใช้คิวรี่ย่อยเพื่อกำหนดคอลัมน์ของคำสั่ง SELECT หลัก

    ผลลัพธ์ที่ได้จากการคิวรี่ย่อยสามารถนำมาใช้เป็นคอลัมน์หนึ่งในคำสั่ง SELECT ของคิวรี่หลัก ซึ่งผลลัพธ์ที่ได้จากคิวรี่ย่อยหนึ่งประโยคต้องมีค่าเพียงหนึ่งแถว เรียกว่า “Scalar Subquery” ถ้าผลลัพธ์ของคิวรี่ย่อยมีมากกว่า 1 ค่า จะไม่สามารถแสดงผลลัพธ์ของการคิวรี่ได้

Syntax:
SELECT column1, column2, …., (SELECT somevalue FROM table2) FROM table1

การใช้คิวรี่ย่อยในประโยค WHERE โดยใช้ Operator (=, <, >, <=, >=, !=)

    การใช้ Operator ในการเปรียบเทียบนั้นสิ่งสำคัญที่ควรรู้คือ ผลลัพธ์ที่อยู่ในคิวรี่ย่อย ต้องเป็นผลลัพธ์ที่ถูก SELECT ออกมาแล้วมีค่าเดียว เช่น COUNT, MIN, MAX, AVG เป็นต้น ถ้าเป็นการ SELECT แล้วได้ผลลัพธ์ออกมาหลายค่าจะใช้ตัวดำเนินการในการเปรียบเทียบไม่ได้

Syntax: การใช้คิวรี่ย่อยในประโยค WHERE โดยใช้ Operator
SELECT * FROM table1 WHERE somecolumn = (SELECT somevalue FROM table2);

การใช้คิวรี่ย่อยในประโยค WHERE โดยใช้ IN และ NOT IN

    จะใช้ในกรณีที่ผลลัพธ์ของคิวรี่ย่อยมีมากกว่าค่าเดียว

Syntax: การใช้คิวรี่ย่อยในประโยค WHERE โดยใช้ IN และ NOT IN
SELECT * FROM table1 WHERE somecolumn IN (SELECT somevalue FROM table2);

Step 1.ทดลองใช้คำสั่ง IN โดยต้องการคิวรี่ข้อมูลทั้งหมดในตาราง E7_Employee ที่มี Office_id จาก ตาราง E7_Office ที่ไม่มี Office_name คำว่า “Thailand”

SELECT * FROM E7_Employee WHERE Office_id IN (SELECT Office_id FROM E7_Office WHERE Office_name != “Thailand”);


Step 2. ทดลองใช้คำสั่งด้านล่างสังเกตุผลลัพธ์ที่เปลี่ยนแปลงไป

SELECT * FROM E7_Employee
WHERE Office_id NOT IN (SELECT Office_id FROM E7_Office WHERE Office_name != “Thailand”);


4.บันทึกผลการทดลอง Step ที่ 1 และ 2 การใช้คิวรี่ย่อยในประโยค WHERE โดยใช้ IN และ NOT IN

การใช้คิวรี่ย่อยในประโยค WHERE โดยใช้ ANY, SOME และ ALL

ตัวดำเนินการ ANY กับ SOME

    จะมีความหมายเดียวกันคือ แค่ตรงตามเงื่อนไขของบางตัวที่อยู่ในSOME ก็จะเป็นจริง(ในที่นี้จะใช้แค่ SOME) เช่น {1,2,3,4,5,6} <=S OME({1,3,5} อธิบายได้ว่า มีเลขอะไรบ้างในปีกกาที่1 ที่น้อยกว่าหรือเท่ากับ 1 หรือ 3 หรือ 5 ในปีกกาที่2 ดังนั้นคำตอบที่ได้คือ 1,2,3,4,5

Syntax: ANY กับ SOME
SELECT * FROM table1
WHERE somecolumn = SOME(SELECT somecolumn FROM table2);

Step 1.คำสั่งหาข้อมูลของพนักงานที่อยู่ใน Italy หรือ Thailand โดยใช้ SOME

SELECT * FROM E7_Employee
WHERE Office_id = SOME(SELECT Office_id FROM E7_Office
WHERE (Office_name = “Italy”) OR (Office_name = “Thailand”));

หมายเหตุ

เทคนิคการอ่านชุดคำสั่งจากตัวอย่าง Step 1

เราต้องพิจราณาในคิวรี่ย่อยที่อยู่ในวงเล็บ SOME ก่อน

SELECT Office_id FROM E7_Office WHERE (Office_name = “Italy”) OR (Office_name = “Thailand”));

จะได้ข้อมูล Office_id คือ 3 กับ 4 จากนั้นก็แทนค่ากลับลงไปใน SOME

SELECT * FROM E7_Employee WHERE Office_id = SOME(3, 4);

เห็นแล้วใช่ไหมว่ามันอ่านขึ้นง่ายกว่าเดิมเยอะเลย อย่าลืมนำวิธีนี้ไปใช้ในการอ่านชุดคำสั่งนะครับ


ตัวดำเนินการ ALL


    มีความหมายคือ ต้องตรงตามเงื่อนไขของทุกตัวที่อยู่ใน ALL เช่น {1,2,3,4,5} <=A LL({1,3,5}) อธิบายได้ว่า มีเลขใดบ้างในปีกกา1 ที่มีค่าน้อยกว่าหรือเท่ากับ 1 และ 3 และ 5 ที่อยู่ในปีกกา2 ดังนั้นคำตอบที่ได้คือ 1 เพราะ 1 น้อยกว่าหรือเท่ากับ 1 และ 3 และ 5 แต่ถ้าเกิดเป็น 2 ซึ่ง 2 น้อยกว่าหรือเท่ากับ 5 และ 3 จริง แต่ไม่น้อยกว่าหรือเท่ากับ 1 นั่นเอง จึงไม่ใช่คำตอบที่ถูกต้อง

Syntax: ตัวดำเนินการ ALL
SELECT * FROM table1 WHERE somecolumn < ALL(SELECT somecolumn FROM table2);

การใช้คิวรี่ย่อยในประโยค WHERE โดยใช้ EXISTS

    ตัวดำเนินการ EXISTS ใช้ตรวจสอบผลลัพธ์ในคิวรี่ย่อย ถ้ามี 1 แถวขึ้นไปจะทำให้ประโยคนั้นมีค่าเป็นจริง แต่ถ้าไม่มีผลลัพธ์ใดๆเลยจะให้ค่าเป็นเท็จ

Syntax: การใช้คิวรี่ย่อยในประโยค WHEREโดยใช้ EXISTS
SELECT * FROM table1 WHERE EXISTS (SELECT somecolumn FROM table2);

การใช้ประโยค HAVING ร่วมกับคิวรี่ย่อย

    คำสั่ง HAVING ใช้เพื่อกำหนดเงื่อนไขการกรองผลลัพธ์จากคำสั่ง GROUP BY การใช้คิวรี่ย่อยร่วมกับคำสั่ง HAVING จะใช้เมื่อต้องการเปรียบเทียบค่าในฐานข้อมูลเดียวกัน

Syntax: การใช้ประโยค HAVING ร่วมกับคิวรี่ย่อย
SELECT * FROM table1
WHERE somecolumn operator value
GROUP BY somecolumn
HAVING somevalue > (SELECT somevalue FROM table1);

การใช้ Correlated Subquery ภายในตารางเดียวกัน

    เป็นการใช้คิวรี่ย่อยในตารางเดียวกับตารางของคิวรี่หลัก สามารถกรองข้อมูลที่มีความซับซ้อนมากขึ้น Correlated Subquery คือ คิวรี่ย่อยที่อ้างอิงไปยังคำสั่ง SELECT ของคิวรี่หลัก ซึ่งชื่อตารางจะซ้ำกับคิวรี่หลัก จึงต้องใช้คำสั่ง Aliase กำหนดชื่อใหม่ให้กับตารางย่อยหรือตารางหลัก

Syntax: การใช้ Correlated Subquery ภายในตารางเดียวกัน
SELECT * FROM table1 AS new1
WHERE somevalue > (SELECT somevalue FROM table1 AS new 2
WHERE new1.somecolumn = new2.somecolumn);

  • ขั้นตอนการทดลอง
    • การใช้คิวรี่ย่อยเพื่อกำหนดคอลัมน์ของคำสั่ง SELECT หลัก
    • การใช้คิวรี่ย่อยในประโยค WHERE โดยใช้ Operator
    • การใช้คิวรี่ย่อยในประโยค WHERE โดยใช้ IN และ NOT IN
    • การใช้คิวรี่ย่อยในประโยค WHERE โดยใช้ ANY, SOME และ ALL
    • การใช้คิวรี่ย่อยในประโยค WHERE โดยใช้ EXISTS
    • การใช้ประโยค HAVING ร่วมกับคิวรี่ย่อย
    • การใช้ Correlated Subquery ภายในตารางเดียวกัน
  • คำถามท้ายการทดลอง
