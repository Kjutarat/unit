ใบงานการทดลองที่ 4

เรื่อง DRC(Data Retrieval Command) คำสั่ง SQL ใช้สำหรับเรียกดูข้อมูล

ความรู้เบื้องต้น

วัตถุประสงค์ในการทดลอง

  1. เพื่อให้ผู้เรียนสามารถเรียกดูฐานข้อมูลทั้งหมดได้
  2. เพื่อให้ผู้เรียนสามารถเรียกข้อมูลและตั้งชื่อคอลัมน์ได้
  3. เพื่อให้ผู้เรียนสามารถเลือกดูฐานข้อมูลโดยกำหนดคอลัมน์ได้
  4. เพื่อให้ผู้เรียนสามารถเรียกดูฐานข้อมูลโดยระบุเงื่อนไขได้
  5. เพื่อให้ผู้เรียนสามารถเรียกดูฐานข้อมูลที่ซ้ำซ้อนได้
  6. เพื่อให้ผู้เรียนสามารถเรียกดูฐานข้อมูลที่จัดกลุ่มได้
  7. เพื่อให้ผู้เรียนสามารถเรียกดูฐานข้อมูลที่จัดกลุ่มและกำหนดเงื่อนไขได้
  8. เพื่อให้ผู้เรียนสามารถเรียกดูฐานข้อมูลที่เรียงลำดับได้
  9. เพื่อให้ผู้เรียนสามารถเรียกดูฐานข้อมูลที่จำกัดจำนวนในการแสดงผลลัพธ์ได้

ทฤษฎีก่อนการทดลอง


DRC (Data Retrieval Command) คือคำสั่ง SQL ที่ใช้สำหรับเรียกดูข้อมูลโดยใช้คำสั่ง SELECT โดยการกำหนดเรียกดูข้อมูลตามเงื่อนไขต่างๆ(WHERE) เช่น การเลือกเฉพาะคอลัมน์ข้อมูลที่ต้องการ การจัดกลุ่มข้อมูล โดยมีเงื่อนไขในการกำหนดกลุ่มข้อมูล (GROUP BY และ HAVING) การคัดเลือกข้อมูลที่มี การซ้ำซ้อนกัน (DISTINCT) การเรียงลำดับข้อมูลจากมากไปน้อย หรือจากข้อมูลน้อยไปมาก หรือการจำกัดจำนวน ของคอลัมน์ข้อมูลที่ต้องการให้แสดงผล(LIMIT) เพื่อทำให้ได้ข้อมูลที่ต้องการเข้าถึงแบบต่างๆตรงตามที่ผู้ใช้งาน ต้องการค้นหา เนื่องจากฐานข้อมูลมีขนาดที่ใหญ่และซับซ้อนมากเราจึงต้องระบุเงื่อนไขในการเข้าถึงเพื่อข้อมูล ที่ถูกต้อง

รูปแบบการใช้งานคำสั่ง SELECT

SELECT * columname ,
FROM tablename
[WHERE conditions]
[GROUP BY group
[HAVING group_condition]]
[ORDER BY sort_coulumns]
[LIMIT limits];
หมายเหตุ

ข้างหลัง SELECT สามารถใส่ตัวเลือกข้อมูลได้โดย

* หมายถึง เลือกข้อมูลในทุกคอลัมน์ทั้งหมด

Columname หมายถึง เลือกข้อมูลในคอลัมน์ที่ระบุชื่อ

, หมายถึง เลือกข้อมูลในคอลัมน์หลายๆคอลัมน์โดยการใช้ , คั่นและตามด้วยชื่อคอลัมน์

SELECT columnname

FROM tablename
การระบุชื่อตารางว่าต้องการดึงข้อมูลมาจากตารางไหนหากต้องการ ระบุจากหลายตาราง จะใช้เครื่องหมาย , คั่น
[WHERE conditions]
การระบุเงื่อนไขในการดึงข้อมูล ในส่วนนี้หากไม่ระบุ จะหมายถึงการดึงข้อมูล ออกมาทุกเรคคอร์ด
[GROUP BY group
การระบุว่าข้อมูลที่ต้องการแบ่งตามกลุ่มจากคอลัมน์อะไร อาจจะไม่ต้องระบุก็ได้
[HAVING group_condition]]
การระบุเงื่อนไขของการแบ่งกลุ่มข้อมูล ซึ่งคำสั่งนี้จะใช้ร่วมกันคำสั่ง GROUP BY ซึ่งไม่จำเป็นต้องระบุก็ได้
[ORDER BY sort_coulumns]
การระบุว่าให้ข้อมูลที่ต้องการเรียบเรียงออกมาเป็นลำดับหรือไม่ ในส่วนนี้ อาจไม่จำเป็นต้องระบุก็ได้ ซึ่งหมายถึงข้อมูลจะออกมาแบบไม่เรียงกัน
[LIMIT]
การระบุจำกัดจำนวนเรคคอร์ดที่ต้องการให้ข้อมูลแสดงออกมา ไม่จำเป็นต้องระบุก็ได้

ใช้สัญลักษณ์ * เพื่อเรียกข้อมูลทุกคอลัมน์ในตาราง มี syntax ดังนี้

Syntax:
select * from table-name;
หมายเหตุ

- table-name หมายถึง ชื่อตาราง

select * from sales;

Step 2.ทดลองเรียกดูข้อมูลทั้งหมดในตาราง products

select * from products

คำสั่งเรียกดูข้อมูลเฉพาะคอลัมน์ที่ต้องการ

     เป็นคำสั่งที่เอาไว้ใช้สำหรับการเรียกดูข้อมูลในตารางแบบระบุคอลัมน์ที่ต้องการ จะใช้วิธีการระบุชื่อคอลัมน์ที่ต้องการข้อมูล โดยถ้าต้องการหลายคอลัมน์จะใช้เครื่องหมาย , คั่น และหาก ต้องการได้ข้อมูลที่ไม่ซ้ำกันในคอลัมน์จะใช้คำสั่ง DISTINCT

Syntax: เรียกดูข้อมูลเฉพาะคอลัมน์ที่ต้องการ
SELECT column-name FROM table-name;
Syntax: เรียกดูข้อมูลเฉพาะคอลัมน์ที่ต้องการ แบบหลายคอลัมน์
SELECT column-name,column-name FROM table-name;
Syntax: เรียกดูข้อมูลเฉพาะคอลัมน์ที่ต้องการ โดยไม่ให้ข้อมูลซ้ำกัน
SELECT DISTINCT column-name FROM table-name;
Syntax: เรียกดูข้อมูลเฉพาะคอลัมน์ที่ต้องการ แบบหลายคอลัมน์ โดยไม่ให้ข้อมูลซ้ำกัน
SELECT DISTINCT column-name,column-name FROM table-name;
Syntax: เรียกดูข้อมูลเฉพาะคอลัมน์ที่ต้องการโดยเปลี่ยนชื่อคอลัมน์เป็น ตามที่ต้องการกำหนดใหม่
SELECT column-name as new-column-name FROM table-name;
หมายเหตุ

- column-name หมายถึง ชื่อคอลัมน์

- table-name หมายถึง ชื่อตาราง

เรียกดูข้อมูลจังหวัดของตาราง Sales โดยให้ข้อมูลซ้ำกันได้

Step 1.ใช้คำสั่งเพื่เรียกดูข้อมูลเฉพาะคอลัมน์ที่แสดงจังหวัด

SELECT city FROM Sales;

Step 2.เรียกดูข้อมูลจังหวัดของตาราง Sales โดยไม่ให้ข้อมูลซ้ำกัน

SELECT DISTINCT city FROM Sales;

Step 3.เรียกดูข้อมูลจังหวัดและรหัสพนักงานของตาราง Sales โดยแสดงผลไม่ให้ข้อมูลซ้ำกัน

SELECT DISTINCT city,SalesID FROM Sales;

Step 4.เรียกดูข้อมูลจังหวัดของตาราง Sales โดยเปลี่ยนชื่อคอลัมน์เป็น Province

SELECT City as Province FROM Sales;

Step 5.เรียกดูข้อมูลแสดงตำแหน่งจำนวนงาน

Syntax: แสดงจำนวน
SELECT COUNT column-name FROM table-name;
SELECT COUNT Position FROM Sales;

Step 6.ในความเป็นจริงตาราง Sales มี ตำแหน่งงานเพียง 2 งาน แต่คำสั่งข้างต้นนับตำแหน่งงานที่ซ้ำกันด้วย เราจึงใช้คำสั่ง DISTINCT ช่วยในการนับเฉพาะข้อมูลที่ไม่ซ้ำกัน

Syntax: เรียกดูข้อมูลเฉพาะคอลัมน์ที่ต้องการโดยเปลี่ยนชื่อคอลัมน์เป็น ตามที่ต้องการกำหนดใหม่
SELECT COUNT (DISTINCT column-name) FROM table-name;
SELECT COUNT (DISTINCT Position) FROM Sales;

2.จงเรียกดูข้อมูลในตาราง sales โดยต้องการเรียกดูข้อมูล รหัสพนักงาน ชื่อจริง นามสกุล เบอร์โทรติดต่อ เปลี่ยนชื่อเป็น Contact จังหวัด เปลี่ยนชื่อเป็น Province

การระบุเงื่อนไขในการแสดงข้อมูล(WHERE)

     จากการทดลองที่ผ่านมาเราจะเรียกข้อมูลออกมาในรูปแบบคอลัมน์ ซึ่งถ้าหากเราต้องการข้อมูลที่เป็นเรคคอร์ด เฉพาะของข้อมูลใดข้อมูลหนึ่งที่ต้องการจะใช้คำสั่งที่เป็นเงื่อนไขระบุโดยใช้คำสั่ง WHERE แล้วตามด้วยเงื่อนไข หากมีเงื่อนไขที่ต้องการมากกว่าหนึ่งอย่างจะใช้ตัวดำเนินการ(operater) เพื่อการแสดงข้อมูลที่ตรงมากขึ้น

การระบุเงื่อนไขในการแสดงข้อมูล(WHERE)

Syntax:การระบุเงื่อนไขในการแสดงข้อมูล(WHERE)ทั้งหมด

SELECT * FROM table-name WHERE (condition);

Step 1. เลือกแสดงข้อมูลทั้งหมดของพนักงานในตาราง sales ที่มีรหัสพนักงาน s02

SELECT * FROM Sales WHERE SID=’S02’;

3.จงเลือกแสดงข้อมูลทั้งหมดของพนักงานในตาราง sales ที่มีรหัสพนักงานคือ s01


Syntax:การระบุเงื่อนไขในการแสดงข้อมูล(WHERE) แบบเลือกคอลัมน์

SELECT column-name FROM table-name WHERE (condition);

Step 2. เลือกแสดงชื่อและนามสกุลของพนักงานที่มีรหัสพนักงาน s02

SELECT salesID,SFirstName,SLastName FROM Sales WHERE SID=’S02’;

4.จงเลือกแสดงชื่อและนามสกุลของพนักงานในตาราง sales ที่มีรหัสพนักงานคือ s01

การระบุเงื่อนไขในการแสดงข้อมูล(WHERE) โดยใช้ตัวดำเนินการ

Syntax:ตัวดำเนินการวงเล็บ ( )

    เป็นการจัดลำดับความสำคัญโดยการใช้วงเล็บเมื่อมีการใช้ตัวดำเนินการมากกว่าหนึ่งตัวเพื่อให้คำสั่งดำเนินการตามลำดับ ถ้าไม่มีการใส่วงเล็บ
ผลลัพท์ที่ได้ก็อาจจะมีการผิดเพี้ยนไป

SELECT * FROM table-name WHERE (condition);

Step 1. เลือกแสดงข้อมูลทั้งหมดของพนักงานในตาราง sales ที่มีรหัสพนักงาน s02

SELECT * FROM Sales WHERE SID=’S02’;

5.จงเลือกแสดงข้อมูลทั้งหมดของพนักงานในตาราง sales ที่มีรหัสพนักงานคือ s01


Syntax:การระบุเงื่อนไขในการแสดงข้อมูล(WHERE) แบบเลือกคอลัมน์

SELECT column-name FROM table-name WHERE (condition);

Step 2. เลือกแสดงชื่อและนามสกุลของพนักงานที่มีรหัสพนักงาน s02

SELECT salesID,SFirstName,SLastName FROM Sales WHERE SID=’S02’;

6.จงเลือกแสดงชื่อและนามสกุลของพนักงานในตาราง sales ที่มีรหัสพนักงานคือ s01

Step 3. แบบใส่ตัวดำเนินการ ( ) สังเกตุผลลัพท์ที่ต่างกัน

SELECT city,Target sales FROM Sales WHERE (city =’กรุงเทพ’ or city = ‘สมุทรปราการ’ ) AND target sales >= ’30,000’;

7.สังเกตุผลลัพท์ที่ต่างกัน พร้อมบันทึกผลการทดลอง

Syntax:ตัวดำเนินการ NOT

    Step 4. เป็นตัวดำเนินการที่ให้ผลลัพท์ตรงกันข้ามกับคิวรี่โดยสิ้นเชิง โดยใช้สัญลักษณ์ <> หรือ NOT ได้เลย

SELECT SFirst,city FROM Sales WHERE NOT city =’กรุงเทพ’;

Syntax:ตัวดำเนินการ AND

    Step 5. แสดงผลข้อมูลของเซลล์ที่อยู่จังหวัดสมุทรปราการ และมียอดเป้าหมายการขายเท่ากับ 50,000 บาท

SELECT * FROM Sales WHERE city=’สมุทรปราการAND Target sales=’50,000’;

Syntax:ตัวดำเนินการ OR

    Step 6. แสดงผลข้อมูลของเซลล์ที่อยู่จังหวัดสมุทรปราการ หรือมียอดเป้าหมายการขายเท่ากับ 50,000 บาท

SELECT * FROM Sales WHERE city=’สมุทรปราการ OR Target sales=’50,000’;

8.สังเกตุผลการทดลอง NOT AND และ OR บันทึกผลการทดลอง

Syntax:ตัวดำเนินการ BETWEEN

    ใช้สำหรับระบุช่วงของข้อมูลที่ต้องการ

WHERE column-name BETWEEN condition AND condition;
Step 7. แสดงผลชื่อและนามสกุลของเซลล์ที่มีเป้าหมายการขายระหว่าง 20,000 – 200,000 บาท

SELECT SFirstName,SLastName FROM Sales WHERE Target sales BETWEEN ’20,000’ AND ‘2000,000’;

SELECT * FROM Sales WHERE Target sales BETWEEN '30,00'’ AND '50,000';

SELECT SFirstName,SLastName,Phone FROM Sales WHERE Phone like ’01%’;

การจัดกลุ่มของข้อมูล (GROUP BY)

Syntax:GROUP BY

    คือการรวมกลุ่มของข้อมูลที่เหมือนกันไว้ให้จัดอยุ่ในกลุ่มเดียวกัน เช่นการรวมกลุ่มของพนักงานที่ทำงานใน ตำแหน่งต่างๆและ นับจำนวนของแต่ละตำแหน่งงานว่ามีจำนวนกี่คน คำสั่งที่ใช้ในการจัดกลุ่มคือ GROUP BY

GROUP BY column-name ;

Step 1. ต้องการนับจำนวนพนักงานในแต่ละตำแหน่งว่ามีจำนวนพนักงานเท่าไหร่

SELECT Position,COUNT (Position) FROM Sales GROUP BY Position;

12.จงเขียนคำสั่งนับจำนวนพนักงานในแต่ละจังหวัดว่ามีจำนวนพนักงานเท่าไหร่

การจัดกลุ่มข้อมูลแบบกำหนดเงื่อนไข (HAVING)

Syntax:HAVING

    จากการทดลองที่ผ่านมาได้ทำการทดลองจัดกลุ่มข้อมูลแล้ว ยังสามารถกำหนดเงื่อนไขได้อีกด้วย โดยใช้ คำสั่ง HAVING จะใช้กับข้อมูลที่ถูกจัดกลุ่มแล้วเท่านั้น ใช้คู่กับคำสั่ง GROUP BY ใช้ WHERE ไม่ได้

GROUP BY column-name HAVING (condition) ;

Step 1. ต้องการข้อมูลตำแหน่งและนับตำแหน่งจากตาราง Sales โดยจัดเรียงโดยตำแหน่ง ที่มีตำแหน่ง = 4

SELECT Position,COUNT (Position) FROM Sales GROUP BY Position HAVING COUNT (Position) =’4’;

13.บันทึกผลการทดลอง

การเรียงลำดับข้อมูล(ORDER BY)

Syntax:ORDER BY

    ตามปกติถ้าเราไม่ใช้คำสั่ง ORDER BY ข้อมูลที่แสดงผลออกมาจะจัดเรียงตามในฐานข้อมูลโดยอัตโนมัติ แต่ถ้า ใช้คำสั่ง ORDER BY เราสามารถกำหนดให้ข้อมูลเรียงลำดับจากค่าน้อไปมากย (ASC) หรือ มากไปน้อย(DESC) โดยกำหนดต่อจากหลังชื่อคอลัมน์ได้เลย

ORDER BY column-name ASC or DESC;
หมายเหตุ

ASC หมายถึง เรียงข้อมูลจากน้อยไปมาก

DESC หมายถึง เรียงข้อมูลจากมากไปน้อย

Step 1. เรีกดูข้อมูลทั้งหมดจากตาราง Sales โดยจัดเรียงข้อูลจากคอลัมน์ชื่อจริง มากไปน้อย

SELECT * FROM Sales ORDER BY SFristName DESC;

Step 2. เรีกดูข้อมูลทั้งหมดจากตาราง Sales โดยจัดเรียงข้อูลจากรหัสพพนักงาน น้อยไปมาก

SELECT * FROM Sales ORDER BY SID ASC;

Step 3. เรีกดูข้อมูลเฉพาะคอลัมน์ ชื่อและนามสกุล จากตาราง Sales โดยจัดเรียงข้อูลจากคอลัมน์วันที่จ้างงานจาก น้อยไปมาก

SELECT SFirstName,SLastName FROM Sales ORDER BY HireDate ASC;

14.เขียนคำสั่งเพื่อเรียกดูข้อมูลทั้งหมดจากตาราง Sales โดย เรียงตามจำนวนของพนักงานในแต่ละจังหวัดจาก น้อยไปมาก

การจำกัดจำนวนแถว(LIMIT)

Syntax:LIMIT

    จำกัดจำนวนเรคคอร์ดที่ต้องการให้แสดงข้อมูล

SELECT * FROM table-name LIMIT value;

Step 1. จำกัดจำนวนแถวในการแสดงข้อมูลแรก 2 แถว ในตาราง Sales

SELECT * FROM Sales LIMIT 2;
