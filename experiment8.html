ทฤษฎีก่อนการทดลอง


    SQL มีการรองรับตารางได้ 2 ประเภท ได้แก่ตารางฐานข้อมูลจริง ซึ่งได้จากการใช้คำสั่งตาราง (CREATE TABLE จากใบงานการทดลองที่ 2) และตารางเสมือน หรือที่เรียกว่า “วิว” ซึ่งได้จากการใช้ คำสั่งสร้างวิวนั่นคือ (CREATE VIEW)

    

วิว (View)

    คือ ตารางเสมือน( Virtual Table) ในฐานข้อมูลสำหรับจัดเก็บข้อมูลที่ได้ จากการคิวรี่ (Query) คล้ายกับตารางฐานข้อมูลจริงแต่แตกต่างกันที่วิวไม่ได้จัดเก็บฐานข้อมูล ซึ่งคือ วิวเป็นตารางชนิดหนึ่งที่ได้ข้อมูลมาจากตารางอื่นๆหลายๆตารางหรือตารางเดียวด้วยวิธีการตามคำสั่ง ที่คิวรี่ เมื่อข้อมูล ในตารางเหล่านั้นเปลี่ยนไป ค่าข้อมูลในวิวจะถูกเปลี่ยนไปโดยอัตโนมัติ

    SQL Query ที่ถูกจัดเก็บในฐานข้อมูลด้วยชื่อที่กำหนดให้ ใช้เพื่อแสดงผลข้อมูล ในฐานข้อมูลตามวัตถุประสงค์ต่างๆ เช่น การดึงคอลัมน์ที่ต้องการออกจากตารางฐานข้อมูล การดึงแถวข้อมูล การสรุปข้อมูล การหาค่าเฉลี่ย

    วิวมีหลายชนิดจำแนกตามประเภทคำสั่งของการคิวรี่ หรือผลลัพท์ของข้อมูลที่ได้ออกมา
      1.Table Join View คือวิวที่ Join ตารางที่สัมพันธ์เข้าด้วยกันดึงข้อมูลคอลัมน์ที่ต้องการออกมา จากตารางและรวบรวมคอลัมน์ที่เลือกมาทั้งหมดมา
แสดงผลใหม่ในหนึ่งตาราง โดยคิวรี่ที่ใช้วิธีการ Join หลายตาราง เรียกว่า “Base Query” เพราะเป็นพื้นฐานที่ให้คิวรี่ตัวอื่นสามารถเรียกใช้ข้อมูลต่อได้
      2.Row View วิวที่เลือกแถวบางส่วนจากแถวทั้งหมด โดยใช้คำสั่ง WHERE
      3.Field View การเลือกแถวทั้งหมดแต่นำมาเฉพาะบางคอลัมน์เท่านั้น
      4.Filtered Windowed View เลือกแสดงบางคอลัมน์จากแถวที่เลือกไว้โดยใช้คำสั่ง SELECT,WHERE
      5.Summary View คือการเรียกดูข้อมูลแบบสรุปมาแล้วตามเงื่อนไขคำสั่ง QUERY

เงื่อนไขการสร้างวิว


ต้องเป็นคิวรี่ที่ถูกต้องแล้วเท่านั้น

เงื่อนไขการอัพเดทวิว


    - ไม่มี DISTINCT
    - ในประโยคคำสั่ง SELECT จะต้องไม่อ้างถึงคอลัมน์เดียวกันซ้ำๆ(มากกว่าหนึ่งครั้ง)
    - ไม่มี GROUP BY หรือ HAVING
    - ไม่มี UNION(รวมทั้ง EXCEPT หรือ INTERSECT)
    - ประกอบด้วยข้อมูลที่มาจากตารางเดียว
    - แถวที่อัพเดทในวิวจะต้องตรงกับเพียงหนึ่งแถวในตารางฐานข้อมูล
    - ถ้าต้องการเพิ่มข้อมูลแถวใหม่ไปยังวิว วิวนั้นจะต้องมี Primary Key และทุกคอลัมน์ จะต้องกำหนด NOT NULL รวมทั้งไม่มีค่า NULL เป็นค่า DEFAULT

UPDATE RESTRICTION

มาตรฐาน ANSI ได้กำหนดข้อจำกัดที่อาจเป็นสาเหตุให้การอัพเดทไม่เป็นผลสำเร็จ ซึ่งมีเงื่อนไขต่อไปนี้
    - ประโยค SELECT จะต้องไม่มี DISTINCT
    - ประโยค SELECT จะต้องไม่มีฟังก์ชัน SUM
    - ประโยค SELECT จะต้องไม่มีฟังก์ชันทางคณิตศาสตร์
    - ประโยค SELECT จะต้องไม่มีตัวดำเนินการ
    - ประโยค SELECT จะต้องไม่มี ORDER BY
    - ประโยค FROM จะต้องไม่มีหลายตาราง
    - ประโยค WHERE จะต้องไม่มีคิวรี่ย่อย
    - คิวรี่จะต้องไม่มี GROUP BY หรือ HAVING
    - คอลัมน์ที่มีการคำนวณจะไม่สามารถอัพเดทได้
    - คอลัมน์ NOT NULL ที่มาจากตารางหลักจะต้องอยู่ในวิวด้วย เพื่อใช้ INSERT คิวรี่ไปยังฟังก์ชันได้

ขั้นตอนการทดลอง

การสร้าง VIEW

Step 1. สร้างตาราง table1

CREATE TABLE table1
(
Cate_id int(4) NOT NULL ,
Cate_name char(20)
);

Step 2. เพิ่มข้อมูลลงในตาราง table1

insert into table1 (Cate_id,Cate_name) values
('1', 'เครื่องใช้ไฟฟ้า'),
('2', 'เครื่องใช้สำนักงาน'),
('3', 'ยานพาหนะ'),
('4', 'สื่อสาร')
;


Step 3.สร้างตาราง table2

CREATE TABLE table2
(
Prod_id int(4) NOT NULL ,
Prod_name char(20),
Cate_id int(4)
);

Step 4.เพิ่มข้อมูลลงในตาราง table2

insert into table2 (Prod_id,Prod_name,Cate_id) values
('10', 'รถยนตร์','3'),
('11', 'รถบรรทุก','3'),
('12', 'โต๊ะ','2'),
('15', 'คอมพิวเตอร์','1'),
('16', 'เก้าอี้','2'),
('20', 'ถ่าน AAA','4')
;


Step 5. การสร้างวิวจะแสดงผลในรูปแบบของตาราง ข้อมูลจะไม่ได้ถูกจัดเก็บอย่างถาวร โดยการ คิวรี่จะคิวรี่ตารางเดี่ยวหรือใช้วิธีการ Join หลายๆตารางทั้งหมดในฐานข้อมูลก็ได้

Syntax: การสร้างวิว
CREATE VIEW view-name AS
หมายเหตุ

    - View name คือ ชื่อวิว

    1.ทดลองสร้าง BASE VIEW
สร้างประโยค SQL ที่ถูกต้องก่อน

SELECT Cate_id,Cate_name FROM table1;

    2.บันทึกวิวไว้บนระบบฐานข้อมูล โดยใส่ Syntax ไว้ส่วนบนของประโยค SQL ที่ต้องการ

CREATE VIEW table1Details AS
SELECT Cate_id,Cate_name FROM table1;

    3.ใช้คำสั่ง SQL QUERY ตรวจสอบการสร้างวิว

SELECT * FROM table1Details;

Step 6. ทดลองกำหนดชื่อให้กับคอลัมน์ สร้างวิว จาก table2 ชื่อ table2naming โดยให้คอลัมน์ Prod_id มีชื่อเป็น Prodid, Prod_name เป็น Prodname สำหรับผลิตภัณฑ์ ที่อยู่ในหมวดยานพาหนะ

CREATE VIEW table2naming AS SELECT Prod_id as Prodid, Prod_name as Prodname FROM table2 WHERE (Cate_id='3');


Step 7. นำตาราง table1 มา INNER JION กับตาราง table2

CREATE VIEW TABLEJOINVIEW AS
SELECT table1.cate_name, table2.prod_id, table2.prod_name
FROM table1 INNER JOIN table2 ON table1.cate_id=table2.cate_id;

การแก้ไขวิว

    ข้อมูลในตารางจริงจะถูกแก้ไขไปด้วยเหมือนกัน

Syntax: การแก้ไขวิว
UPDATE view-name SET สิ่งที่ต้องการเปลี่ยน WHERE เงื่อนไขดึงข้อมูล;
หมายเหตุ

    - ชื่อวิข้อมูลในตารางจริงจะถูกแก้ไขไปด้วยเหมือนกัน

Step 1.ต้องการเปลี่ยนค่า Prod_name ของ “เก้าอี้” เป็น “โซฟา” ในวิว TABLEJOINVIEW

UPDATE TABLEJOINVIEW SET Prod_name ="โซฟา" WHERE Prod_id=16;


Step 2.ใช้คำสั่งเพื่อตรวจสอบข้อมูลในตาราง table2

select * from table2;


Step 3.ตรวจสอบการอัพเดทใน view TABLEJOINVIEW

SELECT * FROM TABLEJOINVIEW;
